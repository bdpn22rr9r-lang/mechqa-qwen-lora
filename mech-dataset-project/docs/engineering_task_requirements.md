# 机械设计专家数据工程任务规范

## 1. 任务目标

阶段 A 建设 200 条训练金样本和 60 条独立评测样本，用于验证机械设计专家模型的数据质量与单卡训练方案。该数据是工程初稿，不是设计签字、产品认证或安全决策依据。

## 2. 角色与责任

- 数据编写者：依据本项目 schema 编写或生成初稿，不得把模型生成内容标记为专家批准。
- 一级审核员：检查题意、字段、单位、表达、证据和重复问题。
- 二级机械工程审核员：检查失效机理、载荷路径、计算方法、材料状态、制造可行性和安全边界。
- 标准审核员：核对标准号、年份、名称、状态、适用范围和相关条款。
- 项目负责人：批准数据版本进入训练；模型或脚本不得自行批准。

## 3. 工程输入最低要求

涉及设计结论时，样本应按问题需要提供或明确缺少以下信息：

- 功能、载荷方向、载荷谱和异常工况。
- 几何尺寸、结构突变、连接和边界条件。
- 材料牌号、供货状态、热处理和表面状态。
- 工作温度、介质、腐蚀、润滑和环境。
- 目标寿命、可靠性等级、失效后果和维护条件。
- 制造能力、检测方法、装配顺序和测量基准。

缺少关键输入时，答案应给出补充清单和校核路径，不得给出确定安全结论。

## 4. 答案最低要求

每条答案至少完成与题目相关的四项：

1. 指出主要失效模式或质量风险。
2. 解释风险产生的工程机理，避免只列名词。
3. 给出可执行的计算、检查、试验或处置步骤。
4. 说明结论依赖的输入、证据和适用边界。

涉及安全时先写能源隔离、停机、泄压、防坠或人员隔离要求。涉及故障诊断时必须区分“可能原因”“验证证据”和“处置条件”。

## 5. 标准与手册引用

- 标准引用必须来自 `reports/standard_registry.json`。
- 必须写完整标准号和年份，不得使用未经登记的简称或猜测编号。
- 标准状态必须为 `current`，并记录核验日期和官方来源。
- 标准只在适用范围内使用；检测方法标准不能代替设计验收要求。
- 机械设计手册中的表格和经验值必须保留版本、页码、材料状态、单位和使用条件。
- 未确认版权或内部授权的手册内容不得复制到公开训练数据。

## 6. 数值与计算

- 具体数值只能来自题目、可追溯证据、已登记标准或经过测试的计算工具。
- 必须保留单位并进行量纲检查。
- 公式应列出适用假设，例如线弹性、小变形、薄壁、稳态或额定寿命定义。
- 经验系数、安全系数、许用应力、粗糙度、硬度和圆角不得凭记忆指定。
- 计算样本必须由独立基准计算复核；语言模型心算不能作为最终证据。

## 7. 数据隔离与去重

- 训练和评测不得共享 `split_group`。
- 同一事故、DOI、零件参数组合或模板改写不得跨训练和评测。
- 批量样本必须在对象、工况、机理和处置上存在实质差异，不能只替换名称。
- 高频前后缀和近似重复必须进入质量报告，由审核员判断是否删除或重写。

## 8. 审核与状态

- 自动生成和模型辅助样本统一使用 `review_status=v3_pending_review`。
- 标准、固定数值、安全、疲劳、材料性能和故障结论类样本必须二级审核 100%。
- 60 条独立评测样本必须二级审核 100%。
- 只有真人审核完成后才可改为 `approved`，并填写审核员和审核记录。
- 任一标准错配、公式错误、单位错误、虚假安全结论均一票退回。

## 9. 阶段 A 完成定义

- 训练金样本严格为 200 条，类别配比符合 `docs/v3_spec.md`。
- 独立评测严格为 60 条，包含高风险、应拒绝和可明确回答三类。
- schema、ID、文本指纹、泄漏、数值、单位、标准台账和文本异常检查全部通过。
- 质量报告明确记录所有样本仍待真人审核，不以 loss 下降替代工程验收。
- 未达到上述条件时不得扩展到 1000 条或启动 8 卡训练。
