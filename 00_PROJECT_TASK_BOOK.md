# AgenticQwen MUSA 开源复现项目任务书

> 版本：V1.0
> 立项日期：2026-07-15
> 执行平台：worker31005 / 8×MTT S5000 / MUSA
> 项目类型：独立开源复现与硬件适配验证工程

## 1. 任务名称

在 MUSA 服务器上分级复现 AgenticQwen 的公开模型、公开数据流水线、
工具交互评测和最小强化学习训练闭环。

## 2. 立项背景

论文使用 Qwen3-8B、Qwen3-30B-A3B、Qwen3-235B 教师模型、合成数据、
行为树任务和 GRPO 风格多轮强化学习。作者公开了 8B/30B checkpoint、
部分合成数据和基于 verl、SGLang 的训练代码。

当前唯一可用硬件不是论文脚本标注的 H100/CUDA，而是 8 张 MTT S5000。
因此本项目必须同时回答两个问题：

1. 作者公开的模型和数据是否能产生论文所述的方向性收益；
2. 作者公开训练栈能否在当前 MUSA 环境形成可验证的最小闭环。

本项目不预设第二个问题一定成功。兼容性失败并形成完整证据，也属于
有效工程结论。

## 3. 总目标

建立一条可审计、可重复、不可夸大结论的 MUSA 复现流程，最终交付：

- 冻结的上游源码、模型、数据和版本清单；
- Qwen3-8B 与 AgenticQwen-8B 的同协议基线对比；
- 公开 AgenticQwen 数据的结构、质量和任务回放报告；
- verl、SGLang、Ray、FSDP、MCCL 在 MUSA 上的兼容性矩阵；
- 最小 GRPO 冒烟结果，或无法闭环的证据化差距报告；
- 全部命令、配置、日志、指标、checkpoint 和错误订正记录。

## 4. 任务范围

### 4.1 必做范围

1. 新项目与原 LightMech 资产完全隔离。
2. 对远端主机、GPU、驱动、容器、磁盘和网络进行只读复核。
3. 冻结作者仓库 commit、模型版本、数据版本和评测仓库版本。
4. 在 MUSA 上验证 Qwen3-8B 和 AgenticQwen-8B 的基础加载与生成。
5. 建立同提示词、同工具、同解码、同评分的对照评测。
6. 验证公开数据的行数、Schema、重复、空值、长度和训练/评测隔离。
7. 回放至少一组完整的模拟用户、模拟工具和 Rubric 评分任务。
8. 分项验证 GRPO 所需组件的 MUSA 兼容性。
9. 兼容门通过后执行一至三步最小 GRPO 冒烟。
10. 形成阶段验收、风险、订正、决策和复盘文档。

### 4.2 条件范围

只有前置阶段通过后才允许：

- 扩大到 1,000 条公开训练样本；
- 扩大 rollout 数量、响应长度和交互轮数；
- 使用 8 卡 MCCL/FSDP；
- 训练后开展 BFCL-V4/TAU-2 子集对比；
- 将方法迁移到机械领域行为树任务。

### 4.3 明确不在首期范围

- AgenticQwen-30B-A3B 训练；
- 两节点或外部 H100 训练；
- 未经批准的云 GPU、外部 API 或外部数据上传；
- 重建论文全部约 10 万条训练数据；
- 复现论文内部工业系统、WebWalker、XBench、GAIA 全量结果；
- 生产部署或连接真实业务写操作；
- 修改或覆盖原 LightMech/Qwen 项目资产；
- 因兼容性困难而静默改用 SFT，并宣称 GRPO 复现。

## 5. 阶段与工作包

### R0：工程冻结

任务：

- 建立本项目文档、目录和命名规范；
- 登记论文、作者仓库、模型、数据和评测来源；
- 定义复现等级和禁止表述；
- 固定所有日志与产物目录。

交付物：

- 本任务书及全部规范文件；
- 初始来源登记、风险登记和决策记录。

通过条件：文档交叉引用一致，MUSA-only 和资产隔离要求明确。

### R1：只读环境验收

任务：

- 核对 hostname、用户、GPU、驱动、MUSA、MCCL、Docker 和磁盘；
- 核对原容器和挂载，仅记录不修改；
- 核对 ModelScope、Hugging Face、GitHub 和 codeload 可达性；
- 保存带时间戳的验收输出。

通过条件：

- 8 张 MTT S5000 可识别；
- 新项目路径可安全创建；
- 与交接单的差异已经记录并处理；
- 无原资产写入行为。

### R2：公开 checkpoint 对比

任务：

- 下载并校验 Qwen3-8B；
- 通过合规渠道取得 AgenticQwen-8B；
- 在新容器内完成非思考模式推理；
- 使用固定样本比较工具选择、参数、拒绝、状态和耗时。

最小规模：

- 20 条格式/工具冒烟题；
- 50 条固定 Agent 子集；
- 至少重复两次确定性评测。

通过条件：两模型均完成评测，原始轨迹和机器可读指标齐全。

### R3：公开数据流水线

任务：

- 获取公开 AgenticQwen-Data；
- 验证公开页面记录的 37,401 行及实际文件哈希；
- 检查字段、重复、空值、超长样本和异常路径；
- 转换为作者训练格式；
- 回放模拟用户、工具和 Rubric；
- 对训练集与固定评测集执行泄漏检查。

通过条件：数据报告通过，至少 10 条任务可完整回放并复算奖励。

### R4：MUSA 兼容门

按顺序验证：

1. Qwen3-8B 单卡前向和生成；
2. 单卡反向和有限值；
3. MCCL 双卡/八卡基础通信；
4. FSDP 参数分片、保存和恢复；
5. Ray 在 MUSA 上的资源发现与调度；
6. SGLang MUSA 构建、Qwen3 生成和张量并行；
7. verl actor/ref/rollout 最小流程；
8. 模拟工具多轮 rollout；
9. 奖励计算和 GRPO advantage；
10. checkpoint 保存、恢复和评测。

任一步失败即停止后续扩大，并形成兼容性差距报告。

### R5：最小 GRPO 冒烟

建议上限：

- 32 至 128 条训练样本；
- rollout_n=2；
- 最大响应 512 至 2,048 token；
- 最大交互 2 至 4 轮；
- 1 至 3 个训练 step；
- 首先单卡或最小可行卡数。

通过条件：

- 无 NaN、Inf、不可恢复 OOM；
- 参数确实发生有限更新；
- checkpoint 可重新加载；
- 同一批验证任务可运行；
- 日志能够证明这是 GRPO 而非 SFT。

### R6：小规模训练与对照

只有 R5 通过后执行：

- 最多 1,000 条训练数据；
- 固定训练前、训练后和官方 checkpoint 三模型对照；
- 至少两个随机种子或重复运行；
- 报告均值、离散程度、失败类别和资源消耗。

通过条件：训练后模型在预注册核心指标上超过同起点基线，且没有
明显安全、格式或通用能力退化。

### R7：阶段结题

汇总：

- 已达到的复现等级；
- 未达到项目标及根因；
- 是否建议扩大到全量 8B；
- 是否建议开展机械领域迁移；
- 是否需要上游 MUSA 补丁或厂商支持。

## 6. 角色责任

| 角色 | 责任 |
|---|---|
| 项目负责人 | 批准范围、外部资源、变更和最终结论 |
| 执行人/代理 | 按 SOP 执行、保存证据、及时停止和记录 |
| 环境管理员 | 提供堡垒机访问、容器权限和 MUSA 支持 |
| 数据负责人 | 来源、许可、质量、泄漏和版本管理 |
| 评测负责人 | 固定题集、评分器、重复运行和结果审查 |

同一执行人可以承担多个角色，但数据生成、训练和最终评分不得仅靠
未经复核的主观判断。

## 8. 最终交付

- 完整代码和配置快照；
- source manifest 和文件哈希；
- 环境 freeze；
- 数据质量与泄漏报告；
- checkpoint 对比报告；
- MUSA 兼容性矩阵；
- 训练日志、指标和 checkpoint；
- 失败样本分类；
- 错误订正、决策和风险关闭记录；
- 结题复盘。

## 9. 项目成功定义

项目成功不等于必须得到论文相同分数。满足以下任一并证据完整，均是
有效成果：

1. 在 MUSA 上完成公开 checkpoint 和数据流程的可重复复现；
2. 在 MUSA 上完成最小 GRPO 闭环；
3. 明确定位无法完成 GRPO 的最小兼容性阻塞点，并提供可复查证据。

只有第 2 项完成，才允许表述为“MUSA 上完成 AgenticQwen 训练链路复现”。
