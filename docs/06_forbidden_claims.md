# 06 禁止表述(防夸大)

> 任务书 §9 与 §4.3 的红线。违反即导致结论失效。

## 严禁的表述

1. **未通过 R5,不得说**:"完成 AgenticQwen 训练链路复现" / "在 MUSA 上完成 GRPO 训练" / "复现了强化学习"。
   - 只有 R5 最小 GRPO 闭环通过(无 NaN/Inf、参数有限更新、checkpoint 可重载、日志证明是 GRPO 非 SFT),才允许此类表述。

2. **不得静默改 SFT 冒充 GRPO**:若兼容性困难改用 SFT,必须明确标注"SFT 路径",不得表述为 GRPO 复现(任务书 4.3)。

3. **不得夸大方向性收益**:公开 checkpoint/数据若仅产生方向性收益,须如实写"方向性收益",不得写"复现论文结果/达到论文分数"。

4. **不得用 LLM-as-Judge 替代执行验证**(除非必要):优先用工具执行成功、最终状态、规则判定等可验证信号;LLM 评审仅作开放式质量辅助且须标注。

5. **不得省略兼容性失败**:R4 任一步失败必须如实报告并出差距报告;"定位不到最小阻塞点 + 可复查证据"是有效结论,不是失败。

6. **不得连接真实业务写操作**:首期不接生产系统、不做真实写操作(任务书 4.3)。

7. **不得混淆资产**:不得把 mechqa/LightMech 的数据/代码当作本项目来源;所有来源须在 `02_source_registry.md` 独立登记。

## 报告中必须区分的措辞

| 允许(证据支撑时) | 禁止(无证据/未达标时) |
|---|---|
| "MUSA 上完成公开 checkpoint 与数据流程复现"(R2/R3 通过) | "完整复现论文" |
| "最小 GRPO 闭环通过"(R5 通过) | "完成训练链路复现"(R5 未通过) |
| "定位到 R4 第 N 步为最小阻塞点" | "完全不可行"(无定位证据) |
| "SFT 路径"(若改用) | "GRPO 复现"(实为 SFT) |
| "方向性收益" | "达到论文分数" |

## 审核机制

- 每份报告(reports/)发布前,对照本文件逐句检查措辞。
- 决策记录(04_decision_log.md)登记任何"措辞放宽"的例外及理由。
