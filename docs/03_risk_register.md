# 03 风险登记(risk register)

> 倒序追加。每项含:描述 / 影响 / 缓解 / 状态(开/关)。

## R-001 MUSA 上 GRPO/RL 栈不可行(最高风险)

- **描述**:论文用 H100/CUDA + verl/SGLang/Ray。当前唯一硬件是 MUSA,而 MUSA 上 Torch SDPA 反向曾有 NaN(需 flash_attn disabled),verl/SGLang/RL 在 MUSA 上几乎无验证。
- **影响**:R4 兼容门可能失败 → R5 GRPO 无法闭环。这正是任务书 §9 第③项"定位阻塞点"要回答的。
- **缓解**:R4 严格按 10 步顺序逐项验证,任一失败即停并出差距报告;不静默改 SFT 冒充 GRPO。
- **状态**:开(待 R4)。

## R-002 测试集 / 任务模板泄漏

- **描述**:训练与评测共享任务模板会导致评测虚高。
- **影响**:对照结论不可信。
- **缓解**:R3 对训练集与固定评测集执行泄漏检查(任务书 R3 通过条件);自建 50 条 Agent 子集独立。
- **状态**:开(待 R3)。

## R-003 资产混淆(违反隔离)

- **描述**:误把 mechqa/LightMech 的数据或代码引入本项目。
- **影响**:违反任务书 4.1.1,污染复现可审计性。
- **缓解**:独立路径/容器/挂载(01_conventions);来源登记独立;旧仓库历史保留但 main 已切换。
- **状态**:开(持续)。

## R-004 教师模型 / 数据合规

- **描述**:作者公开数据/模型的许可与使用边界未确认;商用前须单独确认。
- **影响**:法律与可发布性风险。
- **缓解**:02_source_registry 逐项记录许可;未确认许可不进入可商业版本;只复现、不二次发布权重。
- **状态**:开(待 R0 登记)。

## R-005 历史覆盖(可恢复性)

- **描述**:清空 main 时若误删 git 历史,5419 条旧数据 + 提交记录永久丢失。
- **影响**:旧资产不可恢复。
- **缓解**:已确认采用"删内容保留 git 历史"——任何时候 `git log`/`git checkout 旧commit` 可完整恢复 mechqa-qwen-lora 全部内容。
- **状态**:关(决策已落地,见 04_decision_log D-001)。

## R-006 结论夸大

- **描述**:把"方向性收益"或"SFT 结果"表述为"GRPO 复现成功"。
- **影响**:违反任务书 §9,误导。
- **缓解**:06_forbidden_claims 明令;报告必须区分 SFT/GRPO;只有 R5 通过才允许表述"训练链路复现"。
- **状态**:开(持续)。
