# 人工审核评分标准

本文件是真人审核员(机械工程师)对每条样本打分的依据,也是 `data/rejected/` 退回判定的依据。审核**不只看语言通顺,更要判工程逻辑正确性**。

## 1. 评分维度(每条 0~5 分)

| 维度 | 评分关注点 |
|---|---|
| 专业准确性 | 原理、失效模式、结论是否正确 |
| 风险覆盖 | 是否遗漏重要工程风险 |
| 边界意识 | 是否说明假设和缺失条件 |
| 数值可信度 | 是否存在无依据具体数值 |
| 公式与单位 | 公式、单位换算是否正确 |
| 工程可执行性 | 建议是否具有可操作性 |
| 表达质量 | 是否清晰、结构合理 |
| 安全性 | 是否避免过度承诺/越界签核 |

人工综合评分 ≥ 4/5 方为通过。

## 2. 一票退回条件(出现任意一项直接 reject)

1. **编造标准编号**(引用不存在的 GB/T、ISO、ASTM 等)
2. **编造材料参数**(材料牌号对应的强度/硬度/模量与已知数据矛盾)
3. **无依据给安全系数**(未经计算/标准/工况论证)
4. **公式错误**
5. **单位错误 / 量级明显错误**(如把 GPa 写成 MPa)
6. **把静强度结论当作疲劳寿命结论**(交变载荷题用静强度答)
7. **条件不足时给出确定的安全结论**(缺关键参数仍下"安全/不安全"判断)
8. **引用不存在的检测或制造要求**

被退回的样本进 `data/rejected/`,附 `reject_reason`(对应上述编号 + 说明),`review_status=rejected`。

## 3. 审核分级与比例(对应 task_type)

| 级别 | 必须审核比例 | 涵盖任务 |
|---|---|---|
| **A 级(100%)** | 必须逐条真人审核 | 安全关键设计、强度/疲劳结论、材料与热处理参数、标准规范、FEA 结论、带具体计算值、故障原因判断 |
| **B 级(≥30%)** | 抽样审核 | 机械原理解释、设计评审、主动追问、制造工艺分析 |
| **C 级(≥10%)** | 抽样审核 | 基础术语、简单上下文抽取、普通知识问答 |

## 4. 审核流程

1. 审核员读 `docs/answer_style_guide.md` + `docs/rejection_and_boundary_rules.md`。
2. 逐条按 §1 打分,检查 §2 退回条件。
3. 通过:`review_status=expert_approved`、填 `reviewer`、记录分数到 `reports/manual_review_report`。
4. 退回:移入 `data/rejected/`、`review_status=rejected`、填 `reject_reason`。
5. 聚合:`build_quality_report.py` 统计各 `task_type` 的审核率与均分。

## 5. 关于本批 v0.1-seed 的诚实声明

v0.1-seed 的黄金样本由 Claude(大模型)生成,`review_status=seed_pending_review`。**它们未经真人机械工程师终审,不构成 `expert_approved`。** 在 A 级审核完成前,不得用于产品声明、安全决策或标准合规训练。本批样本的价值是**验证流水线 + 提供风格基准 + 暴露待审问题**,不是直接可用的认证语料。
