# 答案编写规范

本文件规定各类任务的答案结构。**目的:让模型形成稳定的工程行为,而不是套同一个模板。** 答案必须由工程逻辑驱动,不是机械填空。

## 0. 通用原则(所有类型)

1. 先判断"能否直接下结论",再给内容。信息不足时**第一句就说明**,不要先铺一堆再在结尾说"不过信息不足"。
2. 区分**静强度**与**疲劳**——承受交变载荷必须走疲劳校核路径,不能用静强度结论替代寿命结论。
3. 不编造具体数值。圆角半径、粗糙度、硬度、安全系数、疲劳寿命等**只有满足 [rejection_and_boundary_rules.md](rejection_and_boundary_rules.md) 的 4 个条件之一**才能出现。
4. 涉及标准/许用应力/材料参数且输入未给来源时,说明"需查询适用标准或企业规范",并标 `requires_rag: true`。
5. 计算题必须给公式、单位、假设、适用边界。

---

## 1. 设计评审类(design_review / structural_strength / fatigue_failure)

推荐逻辑(按需取舍,不必每条都全有):

1. **当前能否直接下结论**(能/不能,一句话)
2. **关键失效模式**(应力集中、净截面削弱、疲劳、屈曲、磨损…)
3. **建议的校核路径**(先确认什么、用什么方法、校核什么)
4. **制造、工艺和检测要求**(圆角/倒圆、表面完整性、热处理后加工、无损检测)
5. **当前缺少的信息**(列清单)
6. **可给出的初步判断**(在已有信息范围内的、有依据的判断)

### 正例:横向销孔调质轴

> 当前信息不足,不能直接给出强度或寿命结论。
> 1. 失效模式:横向孔导致净截面削弱,孔边应力集中显著,交变弯曲下需按疲劳校核(非静强度)。
> 2. 校核路径:确认孔径/轴径/孔口倒圆、材料与热处理状态、载荷谱;算名义应力与理论应力集中系数,结合疲劳强度缩减系数和表面系数估算寿命。
> 3. 制造与表面:孔口去毛刺倒圆,关注表面完整性;热处理后加工需考虑对表层残余应力的影响。
> 4. 检测:必要时对孔区做磁粉/渗透等表面无损检测。
> 5. 缺失信息:孔径、轴径、材料牌号与热处理状态、弯矩载荷谱、循环次数、目标寿命、适用安全等级。
> 边界说明:圆角半径、粗糙度、硬度、安全系数等具体数值需结合尺寸、载荷、材料状态、工艺能力和适用标准由责任工程师确定。

### 禁止(无依据直接给)

```
圆角半径 2 mm / 粗糙度 Ra 0.8 / 硬度 HRC 45 / 安全系数 1.5 / 疲劳寿命 10^6 次
```

如需表达,应写成:

> 具体数值需结合尺寸、载荷、材料状态、工艺能力和适用标准确定。

## 2. 工程计算类(engineering_calculation)

必须包含 7 要素:

1. 已知条件(明确列出)
2. 所用公式(写出符号形式)
3. 单位换算(显式)
4. 计算过程(代入+中间结果)
5. 计算结果(带单位)
6. 适用假设
7. 未考虑因素 + 是否需要安全系数或进一步校核

**数值输出的前提**:满足下列至少一个,否则禁止输出具体数值:
- 可由题目给定数据直接计算;
- 可从给定上下文中提取;
- 有明确、可追踪的数据来源(记入 `numeric_claims.source`);
- 来自确定性的计算工具。

---

## 3. 信息不足类(info_insufficient)

**不得只回答"信息不足"**,必须说明缺什么、为什么需要。例如判断轴是否安全,至少追问:

```
轴径与结构尺寸 / 材料牌号与热处理状态 / 弯矩扭矩轴向载荷 /
载荷谱与循环次数 / 静载还是交变 / 键槽孔台阶圆角 / 表面状态 /
工作温度与腐蚀环境 / 目标寿命 / 适用标准与安全等级
```

结构:先说"当前无法给出确定结论",再列缺失项,最后说"补充后可进行的校核"。

---

## 4. 标准与材料参数类(requires_rag)

涉及标准编号 / 许用应力 / 安全系数 / 材料强度 / 硬度范围 / 公差等级 / 粗糙度 / 热处理参数,且输入未提供来源时:

> 该数值需查询适用的国家标准、材料状态或企业规范确定,不能仅凭通用模型记忆给出最终设计值。请提供具体标准号或材料认证信息后复核。

并把样本的 `requires_rag` 设为 `true`。

---

## 5. 有限元解释类(fea_interpretation)

覆盖:网格敏感性、边界条件与约束合理性、应力集中区结果判读、单元类型选择、是否需要 submodel、结果是否超出材料许用、与解析解对照。强调"FEA 结果需要结合工程判断,不能只看云图颜色"。

## 6. 故障诊断类(fault_diagnosis)

结构:现象 → 可能原因(按概率/严重度排序) → 排查步骤(可执行、有先后) → 处置建议。区分"根因"与"诱因",区分"需停机检查"与"可在线监测"。

## 7. 工具调用意识类(tool_awareness)

样本应展示"识别到该问题需要工具/检索"的能力,例如:`requires_tool=true`(需 FEA/强度计算)、`requires_rag=true`(需查标准/材料库)。答案里可包含"此问题建议调用 X 工具/检索 Y 标准"的判断,而非强行作答。

## 8. 风格继承

继承 mechqa-qwen-lora v2 的"边界声明"规范:对会引诱模型编造数值的设计题,output 末尾保留一句固定边界声明(见 [rejection_and_boundary_rules.md](rejection_and_boundary_rules.md) §3 模板)。
