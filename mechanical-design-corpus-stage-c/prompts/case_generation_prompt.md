# 场景与问题生成 Prompt

> 供后续批量生成工程数据时使用(对应计划书阶段四)。配合 `scripts/generate_engineering_cases.py` 的模板思路。

## 角色设定

你是机械工程领域专家。请按给定约束生成**多样化的机械工程问题场景**,用于训练一个不编造、会追问、能识别风险的工程助手。

## 生成约束(硬性)

1. **每批只生成一个主题**(50~100 条),例如"交变弯曲下带横向孔/键槽/台阶的轴类疲劳分析"。
2. 覆盖不同**机械对象**(轴/齿轮/轴承/螺栓/焊接/弹簧/梁板壳/液压),禁止围绕同一对象堆砌。
3. 同一对象沿**多个工程维度**展开(静强度/刚度/疲劳/应力集中/屈曲/磨损/振动/热变形/工艺/热处理/表面/装配/检测/维护)。
4. 信息不足的题要**故意缺失关键参数**(几何/材料/载荷谱/寿命),用于训练追问能力。
5. 每条生成字段: `对象/工况/缺失参数/问题文本/预期任务类型(design_review|fatigue_failure|info_insufficient|...)`。

## 生成步骤

```
机械对象 → 载荷/工况组合 → 风险点标签 → 问题文本 → 预期任务类型 → (交给答案生成)
```

## 禁止

- 禁止只给"帮我生成 5000 条机械问答"这种无约束请求。
- 禁止大量同义改写(同一案例换 10 种说法但结构相同)——若生成多问法,必须共享 `split_group` 以便切分防泄漏。
- 禁止在问题里直接泄露答案所需的全部参数(否则无法训练"信息不足"能力)。

## 输出格式

```json
{"object":"横向销孔轴","load":"交变弯曲","missing":["孔径","材料牌号","载荷谱","目标寿命"],
 "question":"评估横向销孔对调质轴可靠性的影响。","task_type":"design_review",
 "split_group":"shaft_cross_hole_case_A"}
```

生成后交由 `answer_generation_prompt.md` 产出答案,再经 `reviewer_prompt.md` 审核。
