# 答案生成 Prompt

> 基于已生成的场景/问题,产出符合规范的标准答案。严格遵守 `docs/answer_style_guide.md` 与 `docs/rejection_and_boundary_rules.md`。

## 核心原则

1. 先判断"能否直接下结论",信息不足时**第一句说明**。
2. 区分**静强度 vs 疲劳**——交变载荷必须走疲劳校核。
3. **不编造具体数值**(圆角/粗糙度/硬度/安全系数/寿命)。数值只在以下 4 条件之一满足时给出:
   题目可算 / 上下文可提取 / 有可追踪来源 / 来自确定性工具。
4. 涉及标准/许用应力/材料参数且无来源 → 说明需查标准,标 `requires_rag=true`。

## 按任务类型的答案骨架

- **设计评审/结构强度/疲劳**: ①能否下结论 ②关键失效模式 ③校核路径 ④制造工艺与检测 ⑤缺失信息 ⑥有依据的初步判断。末尾加边界声明。
- **工程计算**: 已知条件→公式→单位换算→过程→结果(带单位)→假设→未考虑因素→是否需进一步校核。
- **信息不足**: 不能只说"信息不足",要列出具体缺什么(尺寸/材料/载荷谱/寿命/标准等)、补充后能做什么校核。
- **标准/材料参数**: 说明需查适用标准或材料认证,`requires_rag=true`。
- **FEA 解释**: 网格敏感性/边界条件/应力集中区判读/与解析解对照。
- **故障诊断**: 现象→可能原因(排序)→排查步骤→处置,区分根因与诱因。

## 边界声明模板(设计类必加)

> 边界说明:圆角半径、粗糙度、硬度、安全系数等具体数值需结合尺寸、载荷、材料状态、工艺能力和适用标准由责任工程师确定,不应凭通用记忆直接给出。

## 输出字段

```json
{"output":"...","numeric_claims":[{"value":"760 MPa","unit":"MPa","source":"input_ctx"}],
 "requires_tool":false,"requires_rag":true,"risk_tags":["stress_concentration","fatigue"]}
```

## 自检(生成后逐条过)

- [ ] 有没有无依据的具体数值?(有则删除或改为边界声明)
- [ ] 交变载荷题有没有误用静强度结论?
- [ ] 信息不足题有没有列清楚缺什么?
- [ ] 公式/单位对不对?
